Check the rig is healthy

02 / IDENTITY

    
Say who is calling

    
In a deployed environment APIM derives this from your token. Here you state it; the gateway normalises it and signs it into X-Gateway-Assertion. The API trusts the signature, never the raw headers. 03 / PRODUCT

    
Add a product to the catalog

    
POST /api/v1/products is a synchronous write: validate → EF Core insert → 201 with the new id. This becomes the line item the order in step 05 reserves stock against. Needs products:write and mfa in the signed assertion.

05 / ORDER

    
Place an order — the async pipeline

    
POST /api/v1/orders reserves stock and writes the order row and an OrderCreated event into the outbox in one commit, then returns 201. A background processor publishes the event to Service Bus; the Functions consumers (order-confirmation email, inventory reservation) pick it up off their topic subscriptions. Then drive the order forward: PUT /api/v1/orders/{id}/status raises an OrderStatusChanged event the same way.

06 / END

    
That's the pipeline

    
Signed identity in, verified by the API in Required mode; synchronous writes for catalog and customers; a transactional outbox that publishes order events to Service Bus topics; idempotent Functions consumers off those subscriptions. Payloads captured throughout. Keep going in Scalar (same origin — try-it just works) or the Aspire dashboard for traces.
